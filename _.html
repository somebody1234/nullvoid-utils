what is _.js?

a component framework

yes, it's just an offbrand react/vue/svelte/angular

vanilla js

_.js uses vanilla js, meaning no compile step. It can even be used on static websites!

events


NOTE: _.js makes its own events

what isn't _.js?

backwards compatible

_.js uses some very new js, html and css features, meaning it isn't supported on 100% of browsers.
